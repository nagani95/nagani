# Nagani Room Environment / Palace Atmosphere Production Plan

Status: Draft v0.1  
Project: Nagani Traditional  
Primary Room: ၆ ကောင်ဂျင် / ဂလုန်းဂလုန်း

## 1. Purpose

The final room environment should make players feel like they are inside a premium Myanmar traditional dice room.

The environment must support:

- trust
- realism
- cultural feeling
- dice focus
- mobile-first clarity
- premium traditional atmosphere

The room should not feel like a slot machine, arcade cabinet, or cheap browser game.

## 2. Current Room Status

Current `/six-animal` room is accepted as a stable prototype baseline.

Current accepted state:

- muted room
- 3D-only dice table
- visible physical dice source of truth
- compact betting sheet
- clean result board
- clear settlement layer
- warm dark red / gold atmosphere
- mobile portrait layout passes
- full room flow QA passed

## 3. Final Environment Direction

Final room atmosphere should feel like:

- royal Myanmar interior
- dark lacquer walls
- warm lantern/fire glow
- gold ornament details
- premium traditional table area
- controlled shadows
- deep room background
- focused dice table spotlight

The environment should feel expensive, calm, and trustworthy.

## 4. What The Environment Must Not Do

Do not add:

- slot reels
- casino cabinet feeling
- neon arcade atmosphere
- emoji decoration
- cartoon props
- distracting animated background
- fake gambling noise
- clutter that hides dice
- UI that fights the physical table

The dice table remains the hero.

## 5. Mobile-First Rule

The final environment must work first on iPhone portrait.

Rules:

- dice must remain readable
- betting sheet must remain readable
- result board must remain readable
- room background must not reduce contrast
- decoration must not crowd the screen
- table must stay visually centered
- overlays must not hide dice result

## 6. Environment Asset Layers

Future production environment can be built in layers:

1. background room image or 3D backdrop
2. table spotlight / warm glow
3. side ornaments
4. subtle wall texture
5. gold frame accents
6. optional depth shadows
7. optional host/dealer position later

Each layer must be added carefully without breaking dice focus.

## 7. Short-Term Decision

Do not build a full palace room yet.

Current decision:

- keep existing atmosphere foundation
- plan final environment first
- avoid heavy 3D room work until table/dice production path is stable
- use lightweight visual layers before large environment assets

## 8. Future Acceptance Checklist

Room environment is production-ready only when:

- dice remains the main focus
- mobile portrait is clean
- table feels inside a premium room
- atmosphere supports trust
- no visual clutter appears
- no slot/cabinet feeling appears
- build passes
- full room flow QA passes

## 9. Environment Asset Layer Requirements

The final room environment should be built in safe visual layers. Each layer must support the dice table instead of competing with it.

### 9.1 Background Room Layer

Purpose:

- create depth behind the table
- suggest a premium Myanmar traditional room
- support dark red / lacquer / gold atmosphere

Requirements:

- dark enough to keep dice readable
- no busy pattern behind result board
- no bright object behind dice
- no slot machine/cabinet shape
- no neon arcade style

### 9.2 Table Spotlight Layer

Purpose:

- guide player attention to the dice table
- make the dice result feel important
- create premium stage feeling

Requirements:

- warm amber glow
- soft center focus
- not too bright
- does not wash out dice face textures
- does not hide table borders

### 9.3 Side Ornament Layer

Purpose:

- add cultural/traditional feeling without crowding the screen

Allowed:

- subtle gold frame lines
- dark lacquer side panels
- soft pillar silhouettes
- small ornamental patterns

Not allowed:

- large distracting statues
- animated props near dice
- high-contrast decorations beside result board
- anything that looks like slot-machine side art

### 9.4 Wall Texture Layer

Purpose:

- reduce flat black background feeling
- make the room feel physical

Requirements:

- subtle texture only
- low contrast
- dark red / brown / black range
- must not reduce UI readability

### 9.5 Depth Shadow Layer

Purpose:

- make table feel inside a real room
- separate foreground table from background

Requirements:

- soft shadow behind and below table
- no heavy black area that hides dice
- no fake spotlight that feels theatrical or arcade-like

### 9.6 Optional Host / Dealer Layer

This is not a current priority.

If added later:

- host must not cover dice
- host must not become the main focus
- host should feel traditional and respectful
- host should be optional and removable
- no voice/audio dependency yet

### 9.7 Asset Layer Acceptance Checklist

An environment layer is accepted only when:

- dice remains the clearest object
- betting sheet stays readable
- result board stays readable
- table still feels centered
- mobile portrait stays clean
- no slot/cabinet feeling appears
- build passes
- full room flow QA still passes

- dark red lacquer
- warm gold trim
- royal interior
- soft lantern glow
- respectful traditional atmosphere
- physical dice table focus
- calm trusted live room
- mobile game room composition

### 10.2 Avoid Keywords

Do not guide assets toward:

- slot machine
- casino cabinet
- neon arcade
- cartoon casino
- emoji game
- plastic toy room
- sci-fi gambling room
- cheap browser game
- Las Vegas stage

### 10.3 Background Image Direction

If generating a 2D background image, request:

- dark royal Myanmar interior
- table area left empty in center/front
- soft warm light from top or sides
- subtle gold ornament
- deep shadow behind the table
- no characters in first version
- no text
- no dice in the background image

If building a 3D room later, start simple:

- back wall
- side wall hints
- subtle pillars
- dark lacquer material
- gold trim
- warm light sources
- shadow plane under table

Do not build a heavy full palace before dice/table production assets are ready.

### 10.5 First Asset Prompt Draft

### 10.6 Acceptance Rule

The environment asset is accepted only if it makes the table feel more real while keeping the dice easier to see.

## 11. Environment Implementation Phases

Room environment upgrades must be added carefully. The current `/six-animal` room is stable, so environment work should not disturb dice, table, betting, or result flow.

### 11.1 Integration Rule

Do not replace the room atmosphere all at once.

Every environment upgrade must preserve:

- visible physical dice source of truth
- dice readability
- betting sheet readability
- result board readability
- settlement clarity
- mobile portrait layout
- full room flow stability

### 11.2 Phase 1 — Lightweight Background Upgrade

First production environment upgrade should be a better 2D or layered background.

Scope:

- replace or improve current dark lacquer background
- keep table area visually clear
- add subtle depth behind the table
- avoid characters and heavy props

Acceptance:

- table feels more inside a room
- dice remains the main focus
- no UI readability loss
- build passes

### 11.3 Phase 2 — Table Spotlight / Glow Polish

After background is safe, improve lighting feel around table.

Scope:

- subtle warm table glow
- soft amber focus
- better depth shadow
- no strong moving light

Acceptance:

- dice face is still readable
- table looks more premium
- no arcade/neon feeling

### 11.4 Phase 3 — Side Ornament / Frame Layer

After spotlight is safe, add subtle side-frame atmosphere.

Scope:

- dark lacquer side panels
- thin gold frame accents
- soft pillar silhouettes
- very low contrast ornament

Acceptance:

- no crowding on iPhone portrait
- no distraction from dice
- result board remains readable

### 11.5 Phase 4 — Optional Host / Dealer Planning

Host/dealer is not current priority.

Only consider later after:

- dice/table production quality is stable
- room environment is stable
- mobile layout still has enough space
- character does not hide dice or UI

### 11.6 Phase 5 — Full Environment QA Lock

Final room environment is accepted only after:

- betting phase passes
- bets closed phase passes
- rolling phase passes
- result phase passes
- settlement phase passes
- next round reset passes
- mobile portrait QA passes
- build passes

### 11.7 What Must Stay Untouched During Environment Work

Do not change these during environment visual upgrades:

- dice physics
- result detection
- round timing
- betting logic
- settlement formula
- backend strategy
- sound/audio strategy
- production target correction lock

Environment work must support the stable room, not rebuild it.
